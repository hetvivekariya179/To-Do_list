## 📖 Project Overview

This project is a **To-Do List Mobile Application** developed using the **Flutter framework**.  
The application demonstrates **local data storage** using **SharedPreferences**.

The app allows users to:

- Add tasks  
- View saved tasks  
- Mark tasks as completed  
- Delete tasks  
- Automatically save data locally  

All data is stored on the device and remains saved even after restarting the application.

---

## 🎯 Objective of the Project

The main objective of this activity is:

- To understand local data storage in Flutter
- To implement CRUD operations (Create, Read, Update, Delete)

## 💾 What is SharedPreferences?

SharedPreferences is a Flutter plugin used to store small amounts of data locally in key-value pairs.

In this project:
- Tasks are converted into JSON format
- Stored as a String
- Retrieved and decoded when app restarts

---

## 📱 Application Features

### ✅ 1. Add Task
User can enter a task and add it to the list.

### ✅ 2. View Tasks
All saved tasks are displayed in a ListView.

### ✅ 3. Update Task
User can mark task as completed using checkbox.

### ✅ 4. Delete Task
User can delete task permanently.

### ✅ 5. Persistent Storage
Tasks remain saved even after closing and reopening the app.

## 🔄 CRUD Operations Implementation

| Operation | Description |
|-----------|------------|
| Create | Add new task |

## 🔁 Project Workflow

1. App starts.
2. SharedPreferences loads saved tasks.
3. User enters a new task.
4. Task is added to list.
5. Task list is converted to JSON.
6. JSON data is stored locally.
7. On restart, data is retrieved and displayed.
